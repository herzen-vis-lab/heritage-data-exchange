# Heritage Data Exchange

> **Federated MDM architecture for cross-institutional exchange of digital cultural heritage objects**
> 
> Прототип федеративного MDM-обмена объектами цифрового культурного наследия между университетами, музеями и библиотеками.

Разрабатывается в РГПУ им. А. И. Герцена.

---

## Проблема / Problem

Существующие форматы описания культурного наследия (MARC21, Dublin Core, IIIF, EDM) создавались для **централизованных** каталогов. При переходе к децентрализованной сети независимых организаций они не решают три ключевых задачи:

- **Идентификация узлов сети** — нет механизма уникальной идентификации организации-источника в федерации
- **Record Linkage между узлами** — нет стандартного протокола обнаружения дубликатов и взаимного обогащения записей
- **Reference Data совместимость** — классификаторы и словари локальны; нет механизма согласования авторитетных источников между организациями

Проект исследует **протокол обмена для децентрализованной федеративной сети** как альтернативу единому централизованному каталогу.

---

## Архитектурный подход

Проект применяет паттерны **Federated MDM** к домену цифрового культурного наследия:

| **MDM-паттерн** | **Реализация в проекте** |
|---|---|
| Master Record Identification | `digital_object_guid` (UUID) — уникальный идентификатор объекта в федерации |
| Record Linkage | Механизм обнаружения дубликатов между узлами + двунаправленное обогащение атрибутов |
| Reference Data Management | `Type.authority` (URI) — внешние авторитетные источники для классификации |
| Federated (не Registry) топология | Каждый узел хранит данные локально; нет центрального мастера |
| Data Stewardship | Верификация уровня знаний экспертов-атрибуторов |

Выбор **federated топологии** (против registry/hub-and-spoke) обусловлен требованиями организационной автономности участников и отсутствием доверенного центрального оператора.

## Контекст и публикации

**Область применения:** университеты, музеи, библиотеки — организации с накопленным культурным наследием и потребностью в совместном цифровом каталоге без централизованного оператора.

**Смежные кейсы из enterprise-практики:** задача идентична построению федеративного MDM для холдинговых структур с независимыми юрлицами — та же проблема суверенитета данных, тот же выбор топологии, те же паттерны record linkage.
